Detailed Description

State holder for the current PlayerDto in the application.

This component stores the current player and exposes it via a JavaFX ObjectProperty for ViewModels and Views to bind to. The player is initialized at creation using PlayerService. Updates to the player should go through this state holder to ensure all observers are notified.

Responsibilities:

  • Hold the current PlayerDto in memory.
  • Expose an observable property for UI bindings.
  • Allow manual updates after persistence.

Note: If initialization fails, the application will exit.
